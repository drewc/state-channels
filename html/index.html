State Channels in Javascript

This is a simple way to start interacting with state-channels in the browser.

First we must include the .js in the .html file. If you're viewing this as HTML this has already been done. 

<script src="state-channels.js"></script>

Then open up the inspector and see the console.

The interaction with the State Channels Virtual Machine (SCVM) is contained within the StateChannels global object.

The objects that are shared between the browser, the virtual machine, and potentially other peers and servers are known as Rexpr's.

Micropay

Let's start with a Micropay transaction across three peers.

First import the Micropay constructor. 

const { Micropay } = StateChannels;

The Micropay contructs a Rexpr of the tmicropay type on the SCVM. These Rexpr's represent the shared state of 3 different peer nodes. 

let MP1 = new Micropay(["smith", 10], ["dupont", 10], ["durand", 10]),
    MP2 = new Micropay(["smith", 10], ["dupont", 10], ["durand", 10]),
    MP3 = new Micropay(["smith", 10], ["dupont", 10], ["durand", 10]);

Async Rexpr interaction and Promises

Because the SCVM runs outside of the standard JavaScript VM and interacts with the network almost nothing it does is syncronous.

In fact, even the construction of it is threaded. Thus the first thing we must do is wait on the promise.

Regardless of chosing the async/await approach or the .then continuation any Rexpr object has a $promise property that has a result of the succesful creation in the SCVM of that object.

To interact with the browser let's keep it simple. Copy this over. 

let outId = "#output", stdout = (...things) => {
  things.map(i => typeof i === 'string' ? i : JSON.stringify(i))
    .map(out => $(outId).innerHTML = $(outId).innerHTML + out)
}

Now following the descibed source block below is the actual HTML it describes. 

<h2>Output</h2>
<textarea
  id="output" style="width:100%"
  oninput='this.style.height = "";this.style.height = this.scrollHeight + "px"'></textarea>

So stdout("YAY!\n") does what you expect.

Promise Interaction

Let's look at the promises. First Continuation Passing Style. 

MP1.$promise.then(val => stdout("Was it succesfully constructed? : ", val, "\n"))

And iterative. 

let wasBuilt = await MP2.$promise;
stdout("It was built ok?\n", wasBuilt);

The $ property prefix

There are some properties, like $promise for example, that start with the dollar sign. In general, with the exception of the intruductory one, these are non-promise properties that are "plain javascript" and not an SCVM interaction.

Method Call

All methods return a promise.To call a method we can mostly use the tried and true way. For example a Micropay Rexpr has an lst method that outputs the balance of the accounts to the console.

Let's capture that for demonstration purposes. 

(function () {
    var oldLog = console.log;
    console.log = function (...message) {
      stdout(...arguments, "\n");
      oldLog.apply(console, arguments);
    };
})();

Now our Output textarea may no longer be in view. Move it underneath this section so when scrolling down we see it. 

(_=> {
  let oe = $("#output"), me =$("#methodEnd")
  oe.remove(); me.append(oe); oe.style.height = '6em';
})();

We still do not know if MP3 was properly constructed and we should never run any methods on an invalid object so we'll start with that promise.

Like many methods lst does not return a result. But, we'll use the promise to improve the output as the virtual machine expects a carriage return to complete the output.

There are many times when it is useful to know when something has been finished. 

const { cr } = StateChannels
await MP3.$promise; MP3.lst().then(cr)
